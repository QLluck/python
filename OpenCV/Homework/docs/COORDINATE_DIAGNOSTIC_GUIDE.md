# 🔍 坐标诊断工具使用指南

## 📋 工具说明

我创建了一个专门的诊断工具来全面检查点击坐标映射问题。

### 访问方式

**直接访问：** http://localhost:8000/static/coordinate_diagnostic.html

---

## 🎯 功能特点

### 1. **实时坐标显示**
- 屏幕坐标
- 容器坐标
- 图像坐标
- 原始图像坐标
- 实时状态（图像内/外）

### 2. **详细尺寸信息**
- 原始图像尺寸
- 显示尺寸
- 容器尺寸
- 缩放比例（X/Y）
- 居中偏移

### 3. **视觉反馈**
- 绿色虚线边框（图像边界）
- 可选网格显示
- 点击标记（十字 + 坐标 + 编号）
- 鼠标悬停状态变化

### 4. **操作日志**
- 时间戳
- 详细的坐标转换信息
- 警告和错误提示

## 🧪 使用步骤

### 步骤 1: 打开工具

```bash
# 在浏览器中访问
http://localhost:8000/static/coordinate_diagnostic.html
```

### 步骤 2: 上传图像

- **方式 1:** 点击上传区域，选择文件
- **方式 2:** 拖放图像文件到上传区域

### 步骤 3: 观察尺寸信息

查看右侧面板的"图像尺寸信息"：
```
原始尺寸: 2000 x 1500
显示尺寸: 400 x 300
容器尺寸: 800 x 600
缩放比例: X: 5.000, Y: 5.000
居中偏移: X: 200.0, Y: 150.0
```

**检查点：**
- ✅ X 和 Y 缩放比例应该相同（或非常接近）
- ⚠️ 如果不同，说明图像被拉伸了

### 步骤 4: 移动鼠标测试

移动鼠标到图像上，观察"鼠标位置"信息：

**在图像内：**
```
屏幕坐标: (500, 300)
容器坐标: (300.0, 150.0)
图像坐标: (100.0, 0.0)
原始坐标: (500, 0)
状态: ✅ 图像内
```

**在图像外：**
```
状态: ❌ 图像外
原始坐标: 超出范围
```

### 步骤 5: 点击测试

点击图像的不同位置：

**有效点击：**
- 显示绿色十字标记
- 显示坐标文本
- 显示编号
- 日志记录详细信息

**无效点击（图像外）：**
- 红色边框闪烁
- 日志显示错误
- 不创建标记

### 步骤 6: 分析问题

查看操作日志，寻找：

**正常情况：**
```
✅ 点击 #1: 显示(100.0, 50.0) → 原始(500, 250)
```

**问题情况：**
```
⚠️ 警告: X和Y缩放比例不一致，可能导致坐标偏移
❌ 点击在图像外部，已拒绝
```

## 🔍 问题诊断指南

### 问题 1: 点击位置偏移

**症状：** 点击位置和标记位置不一致

**检查：**
1. 查看"缩放比例"是否一致
2. 查看"居中偏移"是否合理
3. 对比"图像坐标"和"原始坐标"的转换

**可能原因：**
- 图像被拉伸（X/Y 缩放比例不同）
- 偏移计算错误
- 使用了错误的尺寸属性

### 问题 2: 无法点击某些区域

**症状：** 某些区域点击无效

**检查：**
1. 移动鼠标到该区域
2. 查看"状态"是否显示"图像外"
3. 查看绿色虚线边框是否正确

**可能原因：**
- 图像边界计算错误
- 容器尺寸不正确
- CSS 样式影响

### 问题 3: 坐标转换不准确

**症状：** 原始坐标值不正确

**检查：**
1. 验证"原始尺寸"是否正确
2. 验证"显示尺寸"是否正确
3. 手动计算：原始坐标 = 图像坐标 × 缩放比例

**示例计算：**
```
图像坐标: (100, 50)
缩放比例: X: 5.000, Y: 5.000
原始坐标: (100 × 5, 50 × 5) = (500, 250) ✅
```

## 🔧 常见问题

### Q: 为什么显示"超出范围"？

**A:** 鼠标在图像外部（容器的空白区域）

**解决：** 移动鼠标到绿色虚线边框内

### Q: 为什么缩放比例不一致？

**A:** 图像被 CSS 拉伸或压缩

**解决：** 检查 CSS 样式，确保保持宽高比

### Q: 为什么偏移很大？

**A:** 图像在容器中居中显示，容器比图像大

**解决：** 这是正常的，偏移会被正确处理

### Q: 如何验证坐标是否准确？

**A:** 
1. 点击图像的四个角
2. 验证原始坐标是否接近 (0,0), (width,0), (0,height), (width,height)
3. 点击中心，验证是否接近 (width/2, height/2)

## 🎯 下一步

### 如果诊断工具工作正常

说明坐标映射逻辑是正确的，问题可能在主应用的其他地方：
- CSS 样式不同
- 事件处理不同
- 其他 JavaScript 干扰

### 如果诊断工具也有问题

说明是浏览器或图像本身的问题：
- 尝试不同的浏览器
- 尝试不同的图像
- 检查图像格式和尺寸

3. 对比结果
