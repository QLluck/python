# 项目中文注释添加计划

## 目标
为整个项目添加详细的中文注释，让零基础的人也能看懂代码逻辑和算法原理。

## 注释原则

### 1. 分层注释
- **文件级**：模块功能、主要算法、使用场景
- **类/函数级**：详细说明、参数解释、返回值、示例
- **行内注释**：关键步骤、算法细节、为什么这样做

### 2. 通俗易懂
- 用日常语言解释专业术语
- 提供实际例子和类比
- 说明"为什么"而不只是"是什么"

### 3. 循序渐进
- 从简单概念到复杂算法
- 先整体后细节
- 提供学习路径

## 注释模板

### 文件级注释模板
```python
"""
模块名称：XXX 模块

功能概述：
---------
用一两句话说明这个模块是干什么的

为什么需要这个模块？
-------------------
解释这个模块在整个系统中的作用

主要算法/技术：
--------------
- 算法 1：简要说明
- 算法 2：简要说明

实际应用：
---------
在医学图像处理中的具体应用场景

主要函数：
---------
- function1(): 功能说明
- function2(): 功能说明

相关概念：
---------
- 概念 1：解释
- 概念 2：解释

参考资料：
---------
- 论文/书籍链接
- 在线教程链接
"""
```

### 函数级注释模板
```python
def function_name(param1: type1, param2: type2) -> return_type:
    """
    函数功能的一句话概述
    
    详细说明：
    ---------
    更详细的功能描述，包括：
    - 这个函数做什么
    - 为什么需要这个函数
    - 它在整个流程中的位置
    
    算法原理：
    ---------
    如果涉及算法，用通俗语言解释：
    1. 步骤 1：做什么，为什么
    2. 步骤 2：做什么，为什么
    3. ...
    
    参数说明：
    ---------
    param1 : type1
        参数 1 的详细说明
        - 取值范围
        - 默认值
        - 如何选择合适的值
    
    param2 : type2
        参数 2 的详细说明
    
    返回：
    -----
    return_type
        返回值的详细说明
        - 数据格式
        - 取值范围
        - 如何使用
    
    异常：
    -----
    ExceptionType
        什么情况下会抛出异常
    
    使用示例：
    ---------
    >>> # 示例 1：基本用法
    >>> result = function_name(param1, param2)
    >>> 
    >>> # 示例 2：特殊情况
    >>> result = function_name(special_param1, special_param2)
    
    注意事项：
    ---------
    - 注意事项 1
    - 注意事项 2
    
    参考：
    -----
    - 相关函数
    - 参考资料
    """
```

### 行内注释模板
```python
# 步骤 1：做什么
# 为什么这样做？因为...
# 举例：如果输入是 X，这一步会得到 Y
code_line_1

# 步骤 2：做什么
# 这里用到了 XXX 算法/技术
# 原理：...
code_line_2

# 关键点：这里要注意...
# 常见错误：如果不这样做，会导致...
code_line_3
```
