# SVM实验详细计划

## 一、实验概述

本实验旨在通过实践掌握支持向量机（SVM）算法的原理、实现和应用。实验采用两个不同特性的数据集，对比不同核函数的性能，并通过超参数调优提升模型效果。

  - 样本：150个，3类，4特征
  
- **数据集2**：乳腺癌诊断数据集
  - 特点：高维数据，非线性可分
  - 用途：测试RBF核SVM的效果
  - 样本：569个，2类，30特征

**1.3 探索性数据分析**

### 阶段3：模型训练与评估（30分钟）

**3.1 训练三种核函数的SVM**

**线性核（Linear Kernel）**
```python
SVC(kernel='linear', random_state=42)
```
- 适用场景：线性可分数据
- 优点：计算快，可解释性强
- 缺点：无法处理非线性问题

**多项式核（Polynomial Kernel）**
```python
SVC(kernel='poly', degree=3, random_state=42)
```
- 适用场景：多项式可分数据
- 优点：可以捕捉特征间的交互
- 缺点：参数多，容易过拟合

**高斯核/RBF核（RBF Kernel）**
```python
SVC(kernel='rbf', random_state=42)
```
- 适用场景：非线性数据（最常用）
- 优点：泛化能力强，适用范围广
- 缺点：需要调参，计算相对慢

**3.2 模型评估指标**

对每个模型计算：
- **准确率（Accuracy）**：正确分类的比例
- **交叉验证分数**：5折交叉验证的平均准确率
- **混淆矩阵**：查看各类别的分类情况
- **分类报告**：精确率、召回率、F1分数
- **支持向量数量**：了解模型复杂度

**3.3 结果可视化**
- 绘制三种核函数的混淆矩阵对比
- 绘制模型性能柱状图
- 绘制决策边界（2D数据）
- 标注支持向量位置

**输出**：
- 混淆矩阵热力图
- 模型性能对比图
- 决策边界可视化
- 分类报告文本

### 阶段4：超参数调优（20分钟）

**4.1 参数说明**

**C（惩罚系数）**
- 作用：控制对误分类的惩罚程度
- C大：硬间隔，容易过拟合
- C小：软间隔，容易欠拟合
- 搜索范围：[0.1, 1, 10, 100]

**gamma（核函数参数）**
- 作用：控制单个样本的影响范围
- gamma大：影响范围小，容易过拟合
- gamma小：影响范围大，决策边界平滑
- 搜索范围：[0.001, 0.01, 0.1, 1]

**4.2 网格搜索**
```python
GridSearchCV(
    SVC(kernel='rbf'),
    param_grid={'C': [...], 'gamma': [...]},
    cv=5,
    scoring='accuracy'
)
```

**4.3 调优过程**
- 定义参数网格（C和gamma的组合）
- 使用5折交叉验证评估每个组合
- 选择交叉验证分数最高的参数
- 在测试集上验证最佳模型

**4.4 结果分析**
- 绘制参数热力图（C vs gamma）
- 对比调优前后的性能提升
- 分析最佳参数的含义

**输出**：
- 超参数调优热力图
- 调优前后性能对比图
- 最佳参数说明

## 四、关键技术点

### 4.1 为什么要标准化？
SVM使用距离度量，不同量纲的特征会导致：
- 大数值特征主导决策
- 梯度下降收敛慢
- 模型性能下降

### 4.2 如何选择核函数？
- 数据线性可分 → 线性核
- 特征维度高（>样本数）→ 线性核
- 样本数多，特征少 → RBF核
- 不确定 → 先试RBF核

### 4.3 支持向量的意义
- 只有支持向量影响决策边界
- 其他样本可以移除而不影响模型
- 支持向量越少，模型越简单

### 4.4 过拟合与欠拟合
- 过拟合：C太大，gamma太大
- 欠拟合：C太小，gamma太小
- 解决：交叉验证选择合适参数

## 五、预期实验结果

### 5.1 Iris数据集
- 线性核准确率：≈95%
- RBF核准确率：≈97%
- 最佳参数：C=10, gamma=0.1
- 支持向量：约30-40个

### 5.2 乳腺癌数据集
- 线性核准确率：≈96%
- RBF核准确率：≈98%
- 最佳参数：C=10, gamma=0.01
- 支持向量：约100-150个

### 5.3 主要发现
1. RBF核在两个数据集上都表现最好
2. 超参数调优能提升1-3%的准确率
3. 数据标准化对性能影响显著
4. 交叉验证能有效防止过拟合

## 六、可能遇到的问题

### 问题1：中文显示乱码
**解决**：设置支持中文的字体
```python
plt.rcParams['font.sans-serif'] = ['Arial Unicode MS']
```

### 问题2：网格搜索太慢
**解决**：
- 减少参数网格密度
- 使用n_jobs=-1并行计算
- 先粗搜索再细搜索

### 问题3：内存不足
**解决**：
- 减少交叉验证折数
- 使用LinearSVC代替SVC
- 减少参数组合数量

### 问题4：模型性能不佳
**检查**：
- 是否进行了标准化
- 参数范围是否合理
- 数据是否有问题
