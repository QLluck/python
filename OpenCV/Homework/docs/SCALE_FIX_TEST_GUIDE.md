## 📋 测试步骤

### 步骤 1: 打开应用

访问：**http://localhost:8000/static/index.html**

### 步骤 2: 强制刷新

按 **Ctrl+Shift+R** (Windows/Linux) 或 **Cmd+Shift+R** (Mac)

清除缓存，确保加载最新的 JavaScript 代码。

### 步骤 3: 切换到点击模式

点击顶部的 **"点击模式"** 标签。

### 步骤 4: 上传大图像

上传一张 **大于 1280px** 的图像（如 2000x1500）。

这样后端会缩放图像，触发坐标调整逻辑。

### 步骤 5: 打开开发者工具

按 **F12** 打开浏览器开发者工具，切换到 **Console** 标签。

### 步骤 6: 点击图像

点击图像上的某个位置（如中心）。

### 步骤 7: 查看前端日志

在 Console 中应该看到：

```javascript
✅ Image clicked: {
  display: {x: 500, y: 375, w: 600, h: 450},
  actual: {x: 1000, y: 750},
  clamped: {x: 1000, y: 750},
  scale: {x: 2.0, y: 2.0},
  imageSize: {w: 2000, h: 1500}
}

[INFO] 点击位置: (1000, 750) [显示: 500.0, 375.0]
[INFO] 发送 API 请求, max_side: 1280

[SUCCESS] 点击分割完成
```

### 步骤 8: 查看后端日志

在终端中查看服务器日志：

```bash

```

应该看到：

## 🐛 问题排查

### 问题 1: 前端没有显示调整后的坐标

**原因：** 浏览器缓存了旧的 JavaScript

**解决：** 
1. 按 Ctrl+Shift+R 强制刷新
2. 或清除浏览器缓存
3. 或使用隐私模式

### 问题 2: 后端报错"missing required parameter"

**原因：** 后端代码没有重启

**解决：**
```bash
pkill -f uvicorn
cd /Users/jack5/QLluckGithub/python/OpenCV/Homework
bash start_server.sh
```

### 问题 3: 坐标仍然不准确

**检查：**
1. 查看前端日志中的 `originalWidth` 和 `originalHeight`
2. 查看后端日志中的 `scale_factor`
3. 验证 `adjusted_click` 的计算是否正确

**调试：**
```javascript
// 在浏览器 Console 中
console.log('Natural size:', previewOrig.naturalWidth, previewOrig.naturalHeight);
console.log('Display size:', previewOrig.clientWidth, previewOrig.clientHeight);
```
