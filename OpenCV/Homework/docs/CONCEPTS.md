# 医学图像处理概念词典

## 图像基础

### 像素 (Pixel)
- **是什么**：图像的最小单位，就像马赛克的一个小方块
- **例子**：一张 1920x1080 的图片有 2,073,600 个像素
- **在代码中**：`img[y, x]` 表示第 y 行、第 x 列的像素

### 灰度图 (Grayscale)
- **是什么**：黑白图像，每个像素只有一个亮度值（0-255）
- **为什么用**：简化计算，医学图像主要看亮度差异
- **转换**：`gray = cv2.cvtColor(bgr, cv2.COLOR_BGR2GRAY)`

### BGR vs RGB
- **BGR**：OpenCV 的默认格式（蓝-绿-红）
- **RGB**：人类习惯的格式（红-绿-蓝）
- **为什么不同**：历史原因，OpenCV 早期设计如此
- **转换**：`rgb = cv2.cvtColor(bgr, cv2.COLOR_BGR2RGB)`

## 预处理技术

### 中值滤波 (Median Filter)
- **是什么**：去噪技术，用周围像素的中位数替换当前像素
- **类比**：就像"少数服从多数"，如果一个像素和周围差太多，就认为它是噪点
- **效果**：去除椒盐噪声（黑白点）
- **代码**：`cv2.medianBlur(img, ksize=5)`

### CLAHE (自适应直方图均衡化)
- **全称**：Contrast Limited Adaptive Histogram Equalization
- **是什么**：增强图像对比度的技术
- **为什么"自适应"**：把图像分成小块，每块单独增强
- **类比**：就像给照片的不同区域分别调亮度，而不是整体调
- **参数**：
  - `clipLimit`: 对比度限制（2.0 通常效果好）
  - `tileGridSize`: 分块大小（8x8 或 16x16）

### 形态学操作 (Morphology)

#### Top-hat (顶帽变换)
- **是什么**：突出比周围亮的小区域
- **用途**：检测亮的病变（如钙化点）
- **原理**：原图 - 开运算结果 = 小的亮区域
- **类比**：就像用橡皮擦去掉大块的亮区域，只留下小亮点

#### Black-hat (黑帽变换)
- **是什么**：突出比周围暗的小区域
- **用途**：检测暗的病变（如血管）
- **原理**：闭运算结果 - 原图 = 小的暗区域

#### 开运算 (Opening)
- **是什么**：先腐蚀后膨胀
- **效果**：去除小的亮点，平滑边界
- **类比**：就像用橡皮擦擦掉小点，再用画笔描粗边缘

#### 闭运算 (Closing)
- **是什么**：先膨胀后腐蚀
- **效果**：填充小的暗洞，连接断裂
- **类比**：就像用画笔填补小洞，再用橡皮擦修整边缘

## 检测技术

### 阈值分割 (Thresholding)
- **是什么**：把灰度图变成黑白二值图
- **原理**：设定一个阈值，比如 128
  - 像素值 > 128 → 白色 (255)
  - 像素值 ≤ 128 → 黑色 (0)

#### Otsu 阈值
- **是什么**：自动计算最佳阈值
- **原理**：找到一个阈值，让前景和背景的方差最大
- **优点**：不需要手动设定阈值
- **代码**：`cv2.threshold(img, 0, 255, cv2.THRESH_OTSU)`

#### 自适应阈值 (Adaptive Threshold)
- **是什么**：不同区域用不同阈值
- **为什么**：图像光照不均时，固定阈值效果差
- **类比**：就像不同地区用不同的及格线

### ROI (Region of Interest)
- **是什么**：感兴趣区域，就是可能有病变的区域
- **为什么需要**：缩小搜索范围，提高准确度
- **表示方法**：`(x, y, w, h)` - 左上角坐标和宽高

### 连通组件分析 (Connected Components)
- **是什么**：找出二值图中所有连在一起的白色区域
- **用途**：识别独立的病变区域
- **类比**：就像数一张纸上有几个独立的墨点

## 分割技术

### 区域生长 (Region Growing)
- **是什么**：从种子点开始，逐步扩展相似的像素
- **原理**：
  1. 选一个种子点（病变中心）
  2. 检查周围像素是否相似
  3. 相似就加入，不相似就停止
- **类比**：就像水滴在纸上扩散，遇到油就停止

### 分水岭算法 (Watershed)
- **是什么**：把灰度图想象成地形图，从低处"注水"
- **原理**：水会从不同的低点开始上涨，相遇处就是分界线
- **用途**：分离粘连的物体
- **类比**：就像在山谷里注水，水面上升到山脊就停止

## 评估指标

### Dice 系数
- **是什么**：衡量两个区域的重叠程度
- **范围**：0-1，越大越好
- **公式**：`2 * |A ∩ B| / (|A| + |B|)`
- **类比**：就像两个圆的重叠面积占总面积的比例

### IoU (Intersection over Union)
- **是什么**：交并比，也叫 Jaccard 系数
- **范围**：0-1，越大越好
- **公式**：`|A ∩ B| / |A ∪ B|`
- **区别**：比 Dice 更严格（分母更大）

## 常见参数

### kernel_size (核大小)
- **是什么**：滤波器或形态学操作的窗口大小
- **要求**：通常必须是奇数（3, 5, 7, 9...）
- **为什么奇数**：需要一个中心点
- **如何选择**：
  - 小核 (3x3, 5x5)：保留细节，去噪弱
  - 大核 (15x15, 21x21)：去噪强，可能模糊细节

### clip_limit (裁剪限制)
- **用于**：CLAHE 对比度增强
- **范围**：通常 1.0-4.0
- **作用**：防止过度增强产生噪点
- **如何选择**：
  - 小值 (1.0-2.0)：增强温和，噪点少
  - 大值 (3.0-4.0)：增强强烈，可能有噪点

### min_area (最小面积)
- **用于**：过滤小的连通组件
- **单位**：像素数
- **作用**：去除噪点和小的误检
- **如何选择**：根据病变大小，通常 50-500

## 数据类型

### np.ndarray
- **是什么**：NumPy 数组，存储图像数据
- **形状**：
  - 灰度图：`(height, width)`
  - 彩色图：`(height, width, 3)` - 3 个通道
- **数据类型**：
  - `uint8`: 0-255 的整数（最常用）
  - `float32`: 0.0-1.0 的浮点数（计算用）

### base64
- **是什么**：把二进制数据编码成文本
- **用途**：在网络上传输图像
- **优点**：可以直接嵌入 HTML/JSON
- **缺点**：比原始数据大约 33%

## 调试技巧

### 可视化中间结果
```python
# 显示图像
cv2.imshow('窗口名', img)
cv2.waitKey(0)  # 等待按键
cv2.destroyAllWindows()

# 保存图像
cv2.imwrite('output.png', img)
```

### 打印图像信息
```python
print(f"形状: {img.shape}")      # (高, 宽, 通道)
print(f"数据类型: {img.dtype}")  # uint8, float32 等
print(f"最小值: {img.min()}")    # 最暗的像素
print(f"最大值: {img.max()}")    # 最亮的像素
print(f"平均值: {img.mean()}")   # 平均亮度
```

### 常见错误

#### 错误 1：维度不匹配
```python
# 错误：灰度图只有 2 维
gray = cv2.cvtColor(bgr, cv2.COLOR_BGR2GRAY)
print(gray.shape)  # (480, 640)

# 正确：需要转回 3 维才能显示彩色
gray_bgr = cv2.cvtColor(gray, cv2.COLOR_GRAY2BGR)
print(gray_bgr.shape)  # (480, 640, 3)
```

#### 错误 2：数据类型错误
```python
# 错误：某些函数要求 uint8
img_float = img.astype(np.float32) / 255.0
cv2.medianBlur(img_float, 5)  # 报错！

# 正确：转回 uint8
img_uint8 = (img_float * 255).astype(np.uint8)
cv2.medianBlur(img_uint8, 5)  # 正确
```

#### 错误 3：核大小必须是奇数
```python
# 错误
cv2.medianBlur(img, 4)  # 报错！4 是偶数

# 正确
cv2.medianBlur(img, 5)  # 5 是奇数
```

## 常见问题 FAQ

**Q: 为什么要转灰度图？**
A: 简化计算，医学图像主要看亮度差异，颜色信息不重要。

**Q: CLAHE 和普通直方图均衡化有什么区别？**
A: CLAHE 是分块处理，避免整体增强导致某些区域过亮或过暗。

**Q: 为什么核大小必须是奇数？**
A: 需要一个中心点，偶数没有明确的中心。

**Q: Dice 和 IoU 哪个更好？**
A: 看应用场景。Dice 对小目标更友好，IoU 更严格。

**Q: 如何选择合适的参数？**
A: 从默认值开始，观察结果，逐步调整。没有万能参数，需要针对具体图像调优。
