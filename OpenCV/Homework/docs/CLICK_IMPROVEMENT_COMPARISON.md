## 🎨 阈值策略详解

### 新的阈值计算公式

```python
# 1. 根据局部标准差选择基础阈值
if std < 10:
    base_T = 12      # 均匀区域
    std_factor = 0.8
elif std < 20:
    base_T = 15      # 较均匀
    std_factor = 0.7
elif std < 30:
    base_T = 18      # 中等纹理
    std_factor = 0.6
else:
    base_T = 22      # 复杂纹理
    std_factor = 0.5

# 2. 根据边缘密度调整
if edge_density > 0.15:
    edge_factor = 0.9   # 边界清晰
elif edge_density > 0.08:
    edge_factor = 1.0
else:
    edge_factor = 1.1   # 边界模糊

# 3. 计算最终阈值
grow_T = base_T * edge_factor + std * std_factor

# 4. 根据种子点位置微调
if abs(seed_relative) > 0.8:
    grow_T *= 1.2   # 极端区域
elif abs(seed_relative) > 0.5:
    grow_T *= 1.1

# 5. 限制范围
grow_T = clip(grow_T, 12, 50)
```

### 设计理念

1. **更大的基础阈值** - 避免过于保守
2. **更大的标准差系数** - 更好地适应纹理变化
3. **更宽松的范围** - 允许更大的灵活性
4. **保持智能性** - 仍然根据图像特征自适应

## 💡 使用建议

### 何时效果最好

✅ **现在更适合：**
- 渐变物体（中心深，边缘浅）
- 纹理复杂的区域
- 需要完整分割的场景

✅ **仍然适合：**
- 物体与背景有明显差异
- 点击物体中心
- 物体灰度相对连续

### 如果分割仍然不够

**可以尝试：**
1. 点击物体更中心的位置
2. 检查图像预处理是否合适
3. 考虑手动调整阈值（如果有 UI）

### 如果分割过度

**可以尝试：**
1. 点击更接近边缘的位置
2. 使用对比度更高的图像
3. 考虑减小阈值范围

## 📈 性能影响

**好消息：** 阈值增大对性能影响很小


原因：
- floodFill 的速度主要取决于图像大小
- 阈值大小对速度影响不大
- ROI 裁剪和降采样仍然有效
